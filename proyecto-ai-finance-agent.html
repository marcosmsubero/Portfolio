Asistente financiero con IA + automatización en n8n

En este proyecto reconstruí y adapté un asistente financiero basado en agentes IA dentro de n8n, con memoria conversacional, herramientas conectadas a datos reales y capacidad para consultar presupuesto, revisar gastos e insertar nuevos movimientos desde una conversación natural. El valor del proyecto no está solo en el chatbot, sino en la arquitectura completa: orquestación del agente, conexión con herramientas, persistencia de contexto y diseño reutilizable para un caso de uso real.

Qué demuestra

Diseño de un workflow productivo en n8n con enfoque de arquitectura: integración de un agente IA, memoria persistente, herramientas conectadas a datos estructurados y separación clara entre conversación, lógica y almacenamiento.

Caso de uso

El usuario puede consultar su situación financiera en lenguaje natural, revisar gastos por categoría, contrastar gasto real frente a presupuesto disponible y registrar nuevos movimientos sin interactuar directamente con la base de datos.

Valor profesional

El proyecto traduce conceptos de IA aplicada en una solución operativa y presentable: automatización, persistencia, tools, integración cloud y una base clara para escalar hacia asistentes verticales más complejos.

Resumen ejecutivo

El flujo se apoya en un agente IA dentro de n8n que recibe mensajes del usuario desde un chat, interpreta la intención y decide qué herramienta utilizar en función de la consulta. Para mantener continuidad entre mensajes, la memoria conversacional se almacena en PostgreSQL. A nivel de datos, el asistente puede consultar gastos, consultar presupuesto e insertar nuevos registros, lo que lo convierte en un ejemplo práctico de agente conectado a sistemas externos. La solución está planteada con foco en mantenibilidad: el workflow es legible, modular y suficientemente claro como para poder mostrarse en portfolio como proyecto técnico real.

Planteamiento

Objetivo funcional

  • Permitir consultas financieras en lenguaje natural sin exponer la complejidad técnica al usuario.
  • Centralizar lógica conversacional, tools y memoria dentro de una arquitectura reproducible en n8n.
  • Demostrar cómo un agente IA puede operar sobre datos estructurados con utilidad real.

Decisiones técnicas

  • Uso de AI Agent como orquestador principal del flujo conversacional.
  • Persistencia del contexto mediante Postgres Chat Memory.
  • Separación de herramientas para leer gastos, leer presupuesto e insertar movimientos.

Enfoque de portfolio

  • Reconstrucción fiel del workflow original antes de plantear mejoras opcionales.
  • Arquitectura explicable para audiencias no técnicas y suficiente profundidad para perfil técnico.
  • Base sólida para evolucionar hacia una versión más profesional con stack cloud completo.

Arquitectura

La solución está estructurada en capas sencillas: entrada conversacional, razonamiento del agente, memoria persistente y herramientas externas. Esta separación permite explicar con claridad qué hace cada componente y facilita su mantenimiento o futura migración a una arquitectura más avanzada.

Entrada

El flujo comienza con un trigger de chat en n8n. Cada nuevo mensaje del usuario entra al workflow como input del agente.

Razonamiento

El nodo AI Agent interpreta la consulta, evalúa si necesita herramientas externas y delega la acción correspondiente en función del objetivo del usuario.

Persistencia

La memoria conversacional se guarda en PostgreSQL para mantener contexto entre mensajes y permitir una experiencia más coherente.

Datos

Las tools conectan el agente con tablas de gastos y presupuesto, permitiendo lectura de datos e inserción de nuevos movimientos.

Consultas de gastos

El agente puede revisar movimientos registrados y responder preguntas sobre gasto por categoría, volumen acumulado o disponibilidad restante.

Seguimiento de presupuesto

La lógica del asistente permite contrastar presupuesto planificado con gasto ejecutado y devolver una respuesta resumida al usuario.

Inserción de datos

Además de consultar, el agente puede registrar nuevos movimientos, lo que convierte el flujo en un sistema transaccional sencillo y no solo en un visor de datos.

Stack técnico

Orquestación

n8n como plataforma principal de automatización y construcción del workflow.

IA y conversación

AI Agent + modelo LLM para interpretar intención, seleccionar herramientas y generar respuestas útiles.

Persistencia

PostgreSQL / Supabase para la memoria conversacional del agente.

Datos operativos

Airtable como capa de datos estructurados para gastos y presupuestos en la versión original del flujo.

Resultado técnico

El proyecto demuestra cómo pasar de un flujo experimental a una solución explicable, conectada a datos reales y suficientemente ordenada como para ser enseñada como caso práctico.

Lectura profesional

Más allá del componente visual, el interés del proyecto está en la arquitectura: herramientas, memoria, integraciones y una lógica suficientemente clara como para ser mantenida.

Escalabilidad

La base permite evolucionar hacia una versión más robusta con SQL nativo, validaciones adicionales, autenticación de usuario y una capa de reporting financiero más avanzada.

Conclusión

Este proyecto demuestra una aplicación práctica de agentes IA más allá del prototipo conversacional. El valor real aparece cuando el agente se conecta a herramientas, mantiene contexto y puede operar sobre datos estructurados de forma útil. Como pieza de portfolio, resume bien una combinación que me interesa especialmente: automatización, datos, arquitectura cloud e IA aplicada a procesos reales.
